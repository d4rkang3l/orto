Ortoar リファレンス マニュアル

１．概要

Ortoar は

  • クラスファイル
  • VM（バーチャルマシン)
  • リソースファイル。Class.getResourceAsStream(String) で使用します。

を束ねて、一つの HTML ファイルを生成します。

２．使い方

java -jar ortoar.jar [options] pathNames ...

pathName にフォルダ名を指定するとそのフォルダ以下を再帰的に追加します。

３．オプション

引数説明
-main クラス名 【必須】最初に実行するクラスを指定します。
-mainHTML HTMLファイル名 起動時にあらかじめ読み込む HTML ファイルを指定します。
ここで指定するファイルは主にデザインの HTML ファイルです。
具体的な使い方についてはチュートリアルをお読みください。
-compress 圧縮HTMLを生成します。起動が２倍遅くなります。
Orto はWeb サーバーの gzip 圧縮と併用することを前提に作られています。
どうしても gzip 圧縮を有効に出来ない場合に苦肉の策としてご利用ください。
gzip 圧縮だと 1/4 になりますが、Ortoar で圧縮をかけると 1/3 程度です。
-language 言語名 言語を指定してください。日本語はja、英語はenです。
-encoding エンコーディング名 Shift_JIS などのエンコーディングを指定してください。
-scriptEmbedded JavaScript と HTML を結合して JavaScript を埋め込みます。
-outputHTML ファイル名 指定したファイル名で HTML ファイルを作成します。
指定していない場合は -main で指定したクラス名から生成するファイル名を決めます。
-outputScript ファイル名 指定したファイル名で外部 JavaScript ファイルを作成します。
指定していない場合は -main で指定したクラス名から生成するファイル名を決めます。
-version Orto のバージョンを表示します。

４．サーバーへの配備について

Ortoar が生成する HTML ファイルを Web サーバーにアップロードするだけで、 Orto アプリケーションは利用できますが、Web サーバーの gzip 圧縮を有効にすることを 強くお勧めします。Orto は gzip 圧縮が有効であることを前提に作られています。 Apache に組み込む場合は、mod_gzip を組み込み、必要な設定を行ってください。 Microsoft IIS の場合は、gzip 圧縮を有効にするオプションがあります。

ISP（プロバイダー）のホームページサービスを利用するなど、 どうしても、gzip 圧縮を有効に出来ない場合は、Ortoar に -compress オプションを つけてください。起動は遅くなりますが圧縮 HTML を生成します。

５．例

MonkeyTiger.class が実行クラスの場合。MonkeyTiger.html を生成します。

java -jar ortoar.jar -main MonkeyTiger MonkeyTiger.class
